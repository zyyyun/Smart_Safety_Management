# Phase 10 Plan 01 — LAN RTSP 자동 수신 감지

## Goal

PC와 Drift X3 카메라가 같은 네트워크에 있다는 전제에서, 사용자가 본부 PC에서
`demo_rtsp_real_camera.ps1` 또는 별도 pull 스크립트를 실행하지 않아도 Windows service 로
떠 있는 `ai_agent/main.py` 가 카메라 RTSP 수신 가능 상태를 자동 감지하고 기존 YOLO
detection cycle 을 즉시 구동한다.

## Direction Change

이전 Phase 10 초안은 모바일이 RTSP 프레임을 Supabase Storage 로 올리고 본부 PC가
`rtsp_poc_pull.py` 를 polling 하는 frame sampler PoC 였다. 새 사용자 결정으로 Phase 10은
다음 전제를 잠근다.

- PC와 모바일/카메라는 같은 LAN 에 있다.
- 본부/현장 PC 에는 `SmartSafetyAiAgent` Windows service 를 띄운다.
- 사용자는 PC에서 추가 스크립트를 수동 실행하지 않는다.
- 카메라가 WiFi 에 붙어 `rtsp://...:554` 를 열면 agent 가 이를 감지한다.
- 감지 후 기존 `scheduler.run_detection_cycle()` 경로를 그대로 사용한다.

## Implementation Tasks

1. Config
   - `RTSP_AUTODETECT_ENABLED=true`
   - `RTSP_AUTODETECT_INTERVAL_SEC=15`
   - `RTSP_AUTODETECT_TIMEOUT_SEC=1.5`

2. RTSP probe
   - `scheduler._rtsp_probe_url(url, timeout_sec)` 추가.
   - `urlparse()` 로 host/port 파싱.
   - TCP connect 만 확인하고, frame decode 는 기존 `snapshot.capture_rtsp()` 에 맡긴다.

3. Auto-detect scheduler job
   - `run_rtsp_autodetect_cycle()` 추가.
   - `bridge.fetch_active_cameras()` 에서 RTSP URL 카메라만 필터.
   - `unreachable/unknown -> reachable` 전이 시 `run_detection_cycle()` 즉시 호출.
   - fall detector 가 로드되어 있으면 `run_fall_cycle()` 도 같이 호출.

4. Overlap guard
   - 자동 감지 job 과 기존 1분 `general_detection` job 이 겹치지 않도록
     `_detection_cycle_lock` 으로 duplicate cycle 을 막는다.

5. Tests
   - host/port probe 파싱.
   - reachable transition 에서 detection cycle 이 1회만 트리거됨.
   - autodetect disabled 면 DB fetch 도 하지 않음.

## Manual Scenario

1. Drift X3 카메라가 `cameras.live_url_detail = rtsp://192.168.0.13/live` 로 등록되어 있어야 한다.
2. PC에서 AI agent service 를 설치/시작한다.
   ```powershell
   powershell -ExecutionPolicy Bypass -File scripts\ai_agent_service.ps1 -Action Install
   powershell -ExecutionPolicy Bypass -File scripts\ai_agent_service.ps1 -Action Status
   ```
3. 카메라 전원을 켜고 같은 WiFi 에 붙인다.
4. `logs/ai_agent.log` 에 아래 흐름이 찍히는지 확인한다.
   - `rtsp_autodetect: camera_id=... reachable`
   - `rtsp_autodetect: triggering YOLO detection cycle`
   - `=== 일반 감지 사이클 시작`
5. 사람이 카메라 앞에 서면 1분 내 `detection_events` 에 `person` 이벤트가 들어오는지 확인한다.

## Success Criteria

- 별도 demo/pull 스크립트 없이 service 만으로 RTSP 카메라 수신 가능 상태를 감지한다.
- 카메라 reachable 전이 후 다음 probe tick 에 YOLO detection cycle 이 1회 이상 트리거된다.
- 기존 Phase 8 직접 RTSP capture/detection path 를 재사용한다.
- 자동 감지 job 과 기존 detection interval job 이 중복 실행으로 충돌하지 않는다.
