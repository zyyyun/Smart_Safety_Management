PASS 산업안전 데이터셋 라벨링 가이드 (외주 발주용)

주관기관 : 주식회사 애나 (ANNA Inc.)
버전 : v1.0 │ 발행일 : 2026-05-15
용도 : 라벨링 용역사 / 라벨러 작업 지침서

0. 시작하기 전에

0.1 본 가이드는 무엇인가요?

본 가이드는 산업 현장 영상·이미지에서 위험 상황을 찾아내고 표시하는 작업(라벨링) 의 방법을 정리한 문서입니다.

  • 작업 대상 : 공장·물류·건설 현장 사진과 영상
  • 작업 내용 : 위험한 장면에 사각형 박스를 그리고 이름표를 붙임
  • 코딩 지식 필요 없음 — 본 가이드의 표와 예시만 그대로 따라하면 됩니다.

0.2 작업 흐름 한눈에 보기

1) 발주처가 제공한 사진·영상을 받는다
        ↓
2) 라벨링 도구(발주처 지정)에서 파일을 연다
        ↓
3) §1 클래스 표에서 사진에 해당하는 위험 상황을 찾는다
        ↓
4) 화면 속 위험 객체에 사각형(Bounding Box)을 그린다
        ↓
5) 박스에 이름표(라벨)를 붙인다 — 예: fire, person, machine
        ↓
6) "COCO 형식으로 내보내기" 버튼을 클릭한다
        ↓
7) 결과물을 PM(작업 관리자)에게 제출한다

0.3 꼭 지켜야 할 3가지

  1. 단순함 — 박스 그리기 + 이름 붙이기. 이 두 가지만 합니다.
  2. 정확성 — 헷갈리면 §1 클래스 표와 §2 예시를 보세요. 그래도 모르면 PM에게 문의.
  3. 품질 — 박스는 객체에 딱 맞게. 너무 크거나 작으면 재작업.

1. 라벨링 대상 클래스 (전체)

본 발주의 라벨링 대상은 다음 8개 위험 상황 + 1개 정상 상황 입니다.

1.1 클래스 정의 표 (가장 중요)

코드 한글 이름 라벨 스트링 (영문) 위험도 한 줄 설명
AC-001 화재 / 연기 fire, smoke Critical 작업장 안에 불이 났거나 연기가 나는 상황
AC-002 작업자 끼임 entrapment, person, machine Critical 작업자의 팔·다리·몸이 기계에 끼인 상황
AC-003 낙상 / 쓰러짐 fall_collapse, person Critical 작업자가 바닥에 쓰러진 상황 (높이에서 떨어진 게 아님)
AC-004 추락 fall_height, person Critical 작업자가 사다리·발판·지붕 등 높은 곳에서 떨어진 상황
AC-005 지게차 충돌 / 가동구역 진입 forklift_collision, forklift, person, hazardous_zone Warning ~ Critical 지게차가 작업자에게 가까이 왔거나, 지게차 가동 구역에 작업자가 들어간 상황
AC-006 안전모 미착용 no_helmet, person Notice 작업자가 안전모를 안 쓴 상황 (안전모 의무 구역)
AC-007 작업자 건강 이상 health_anomaly, person Warning ~ Critical 작업자가 비틀거리거나 컨디션이 안 좋아 보이는 상황
AC-008 가스누출 / 질식 / 위험구역 진입 gas_leak, person, hazardous_zone Critical 가스센서 경보가 울린 구역, 또는 가스·화학물질 구역에 작업자가 들어간 상황
AC-000 정상 상황 normal — 위험 상황이 없는 평범한 작업 사진. 박스 없이 그냥 정상으로 분류만

1.2 위험도 의미

위험도 의미
Critical 즉각적 인명 위험 — 사람이 다치거나 죽을 수 있는 상황
Warning 사고 발생 가능성 높음 — 곧 위험해질 수 있는 상황
Notice 규정 위반 또는 경미한 위험 — 안전 수칙 위반

1.3 라벨 스트링 사용 규칙

  • 모든 라벨 스트링은 소문자 + 언더스코어(_) 형식
  • 영문 철자 그대로 사용 (한글 라벨 금지)
  • 표 외에 임의 라벨 생성 금지 — 본 표에 없는 스트링은 사용하지 않음

2. 클래스별 상세 가이드 및 예시

각 클래스마다 다음 5가지를 안내합니다.
- 언제 라벨링하나 (상황 정의)
- 무엇에 박스를 그리나 (대상 객체)
- 어떤 라벨을 붙이나 (라벨 스트링)
- 이런 건 라벨링 안 함 (오라벨 방지)
- 이미지 + JSON 예시 (실제 결과물 모양)

2.1 AC-001 화재 / 연기

항목 내용
언제 라벨링하나 사진/영상에 빨강·주황·노랑 불꽃이 보이거나, 회색·검은색·흰색 연기가 보일 때
무엇에 박스를 그리나 (1) 화염이 보이는 영역 / (2) 연기가 보이는 영역 — 둘이 다 보이면 박스를 각각 따로 그림
라벨 스트링 화염은 fire, 연기는 smoke
위험도 Critical

이런 건 라벨링 안 함 (오라벨 방지)

  • 용접할 때 튀는 작은 스파크 (점 같은 불꽃)
  • 흰 김 / 수증기 / 물안개
  • 석양·조명 때문에 빨갛게 보이는 부분
  • 빨간색 옷·표지판

라벨링 예시

[이미지 예시 #1]
공장 내 화재 발생 사진. 빨간 불꽃이 작업대 위에 보이고, 그 위로 회색 연기가 피어오름.
- fire 박스 : 불꽃 영역만 둘러쌈
- smoke 박스 : 연기 영역만 둘러쌈

(실제 이미지는 발주처가 제공)

COCO JSON 결과물 예시

{
  "image_id": 1,
  "file_name": "AC001_fire_0001.jpg",
  "annotations": [
    { "id": 101, "category_id": 1, "category_name": "fire",
      "bbox": [320, 180, 210, 155], "area": 32550 },
    { "id": 102, "category_id": 2, "category_name": "smoke",
      "bbox": [280, 100, 380, 240], "area": 91200 }
  ]
}

2.2 AC-002 작업자 끼임

항목 내용
언제 라벨링하나 작업자 신체(손·팔·다리)가 기계(프레스·롤러·컨베이어 등)에 물려있는 사진
무엇에 박스를 그리나 (1) 작업자 전체 — person / (2) 기계 — machine
라벨 스트링 끼임 상황 자체는 entrapment, 그 안의 객체는 person, machine
위험도 Critical

이런 건 라벨링 안 함

  • 작업자가 의자에 앉아서 휴식 중
  • 작업자가 공구로 일하는 자세 (잠시 정지)
  • 볼트 조이기·정밀 작업 등 의도적으로 가만히 있는 자세

끼임 vs 정상 구별 기준

  • 끼임 : person 박스와 machine 박스가 반 이상 겹쳐있음
  • 정상 : 박스가 가까이 있지만 안 겹침

라벨링 예시

[이미지 예시 #2]
A동 프레스 라인. 작업자 W-017 의 우측 팔이 프레스 기계에 끼인 모습.
- person 박스 : 작업자 전신
- machine 박스 : 프레스 기계 (위에서 누르는 부분)
- 두 박스가 겹친 부분 → 끼임으로 판정

2.3 AC-003 낙상 / 쓰러짐

항목 내용
언제 라벨링하나 작업자가 같은 높이의 바닥에서 쓰러진 사진 (높은 곳에서 떨어진 게 아님)
무엇에 박스를 그리나 쓰러진 작업자 전체
라벨 스트링 fall_collapse, person
위험도 Critical

낙상 vs 추락 구별

  • 낙상 (AC-003) : 같은 바닥에서 쓰러짐. 발이 바닥에 있다가 그 자리에 쓰러짐.
  • 추락 (AC-004) : 사다리·발판·계단 등 높은 곳에서 떨어짐.

이런 건 라벨링 안 함

  • 작업자가 천천히 앉는 자세 (휴식·작업 자세)
  • 도구를 줍기 위해 잠깐 숙임
  • 의도적 바닥 작업 (배관·청소 등)

라벨링 예시

[이미지 예시 #3]
C동 조립 라인 바닥에 작업자가 옆으로 쓰러져 있는 모습.
- person 박스 : 쓰러진 작업자 전신 (가로로 누운 자세)

2.4 AC-004 추락

항목 내용
언제 라벨링하나 작업자가 사다리·발판·고소작업대·지붕 등 높은 곳에서 떨어진 사진
무엇에 박스를 그리나 떨어진 작업자 + (가능하면) 추락 시작 위치
라벨 스트링 fall_height, person
위험도 Critical

이런 건 라벨링 안 함

  • 엘리베이터·리프트로 내려가는 모습
  • 계단을 내려가는 모습 (걸어서)
  • 물건이 떨어지는 모습 (인체가 아님)

추가 표시할 것 — 안전대 착용 여부

사진에 작업자가 안전대(harness)를 안 차고 있는 게 보이면 추가 박스:
- no_safety_harness (안전대 미착용 표시)

라벨링 예시

[이미지 예시 #4]
D동 외벽 발판에서 작업자가 떨어져 지면에 누워있는 모습.
- person 박스 : 지면에 떨어진 작업자
- no_safety_harness : 안전대를 안 차고 있다면 추가

2.5 AC-005 지게차 충돌 / 가동구역 진입

항목 내용
언제 라벨링하나 (A) 지게차가 작업자에게 가까이 다가오는 사진 (3m 이내) OR (B) 지게차 가동 구역(노란 선·경고 테이프 안)에 작업자가 들어간 사진
무엇에 박스를 그리나 (1) 지게차 — forklift / (2) 작업자 — person / (3) 가동 구역 표시 영역 — hazardous_zone (선·테이프로 표시된 구역)
라벨 스트링 상황은 forklift_collision (충돌 위험) 또는 forklift_zone_entry (구역 진입). 객체는 forklift, person, hazardous_zone
위험도 거리 ≤ 3m : Warning / 거리 ≤ 1.5m : Critical

이런 건 라벨링 안 함

  • 지게차가 작업자 옆을 단순 통과 (작업자 방향 안 향함)
  • 정지 중인 지게차 옆에 작업자가 있음
  • 작업허가 받은 작업자의 작업 (안전망 안에서)

라벨링 예시

[이미지 예시 #5]
E동 물류 통로. 지게차 FLT-03 이 작업자 W-028 방향으로 직진 중, 거리 약 1.3m.
- forklift 박스 : 지게차 전체
- person 박스 : 작업자 전체
- 두 박스 거리가 좁음 → 충돌 위험

2.6 AC-006 안전모 미착용

항목 내용
언제 라벨링하나 안전모 의무 구역에서 작업자가 안전모를 안 쓴 사진
무엇에 박스를 그리나 (1) 작업자 머리 영역 — head / (2) 작업자 전체 — person
라벨 스트링 no_helmet, head, person
위험도 Notice (단, 고소작업·화재·가스 동반 시 Warning 격상 — PM 판단)

이런 건 라벨링 안 함

  • 일반 모자·두건만 쓴 경우 (안전모 색상이 아닌 경우)
  • 안전모 의무 구역 밖에서 안 쓴 경우
  • 사진이 너무 어두워서 안전모 여부 확실치 않음

라벨링 예시

[이미지 예시 #6]
A동 Zone-A 입구. 작업자가 안전 조끼만 입고 안전모는 안 쓴 모습.
- head 박스 : 머리 부위만 (안전모가 없는 게 보임)
- person 박스 : 작업자 전체

2.7 AC-007 작업자 건강 이상

항목 내용
언제 라벨링하나 작업자가 비틀거리거나, 휘청거리거나, 사행(좌우 흔들림) 패턴으로 걷는 사진. (쓰러지지는 않았지만 컨디션이 안 좋아 보임)
무엇에 박스를 그리나 작업자 전체
라벨 스트링 health_anomaly, person
위험도 Warning (쓰러지면 AC-003 으로 다시 분류)

이런 건 라벨링 안 함

  • 격렬한 작업 직후 잠깐 쉬는 모습
  • 정상 보행
  • 무거운 물건 들고 균형 잡는 모습

라벨링 예시

[이미지 예시 #7]
F동 조립 라인. 작업자 W-072 가 비틀거리며 걷는 모습. 좌우 균형이 흔들림.
- person 박스 : 작업자 전체 (자세가 불안정해 보임)

2.8 AC-008 가스누출 / 질식 / 위험구역 진입

항목 내용
언제 라벨링하나 (A) 가스누출이 의심되는 시각 단서가 보이는 사진 OR (B) 가스·밀폐공간·화학물질 구역에 작업자가 들어간 사진
무엇에 박스를 그리나 (1) 작업자 — person / (2) 구역 표시 — hazardous_zone / (3) 시각 단서 — 아래 추가 표시 참조
라벨 스트링 상황은 gas_leak (가스누출 의심) 또는 gas_zone_entry (위험구역 진입). 객체는 person, hazardous_zone
위험도 가스 알람 동반 시 Critical / zone 진입 단독 시 Warning

추가 표시할 시각 단서 (보이면 박스)

  • liquid_spill : 액체가 바닥에 흘러있음
  • drum_damage : 드럼통이 찌그러지거나 파손됨
  • colored_plume : 색이 있는 연기·가스 구름
  • missing_respirator : 작업자가 호흡보호구를 안 쓰고 있음
  • ghs_pictogram : 화학물질 위험 표시(GHS 마크) 가 보임

이런 건 라벨링 안 함

  • 용접·도장 작업 시 정상 발생하는 흄/연기
  • 단순 청소 작업 중 사용한 물 자국

라벨링 예시

[이미지 예시 #8]
A동 밀폐탱크 내부. 작업자 W-088 이 호흡보호구 없이 탱크 내부에서 작업 중. 탱크 하부에 액체가 흘러있음.
- person 박스 : 작업자
- missing_respirator 박스 : 작업자 얼굴·코·입 부위 (호흡보호구 없음)
- liquid_spill 박스 : 탱크 하부 액체 영역
- hazardous_zone 박스 : 탱크 내부 전체

2.9 AC-000 정상 상황

항목 내용
언제 라벨링하나 위 8개 위험 상황이 전혀 없는 평범한 작업 사진
무엇에 박스를 그리나 박스 안 그림 (객체 라벨링 없음)
라벨 스트링 이미지 분류만 normal
위험도 없음

정상 사진은 사진 파일을 AC-000_normal/ 폴더에 넣고, 별도 박스 없이 category: normal 만 표기합니다.

3. 클래스별 라벨링 수량 계획 (균등 할당)

PM은 작업 분배 시 다음 비율을 지켜야 합니다. 어느 한 클래스만 많거나 적으면 안 됩니다.

코드 클래스 목표 수량 비율 (전체 대비)
AC-001 화재 / 연기 800장 약 6%
AC-002 끼임 600장 약 4%
AC-003 낙상 / 쓰러짐 800장 약 6%
AC-004 추락 600장 약 4%
AC-005 지게차 충돌 / zone 1,000장 약 7%
AC-006 안전모 미착용 1,000장 약 7%
AC-007 건강 이상 800장 약 6%
AC-008 가스 / zone 1,000장 약 7%
— (위험 상황 소계) 6,600장 약 47%
AC-000 정상 상황 (Negative) 7,000장 약 50%
— (다중 라벨 — §4 참조) 400장 약 3%
합계 약 14,000장 100%

3.1 라벨링 분포 추가 요구사항

각 클래스 내에서도 다음 비율을 지켜야 합니다.

분포 축 권장 비율
시간대 주간 60% / 야간 30% / 교대 전환 10%
조도 주광 40% / 형광등 30% / 저조도 15% / 야간 인공조명 15%
계절·기상 봄·가을 40% / 여름(고온) 30% / 겨울(저온) 20% / 우천 10%
합성 vs 실데이터 실데이터 ≥ 60% / 합성·시뮬레이션 ≤ 40% (AC-002·004·008 은 합성 ≤ 60% 허용)

4. 다중 라벨 (한 사진에 여러 상황이 동시 발생)

한 사진에 위험 상황이 2개 이상 동시 발생한 경우, 각 상황을 모두 박스로 표시합니다.

4.1 다중 라벨 예시

작업자가 안전모 안 쓴 채로 추락한 경우 :
- person 박스 (떨어진 작업자)
- fall_height (추락 상황 라벨)
- no_helmet (안전모 미착용 추가)

4.2 다중 라벨 정리 규칙

  • 활성화된 모든 클래스의 라벨을 모두 표시
  • 박스가 같은 객체(예: person)면 박스는 1개만, 라벨 스트링만 여러 개 부착
  • 다중 라벨 사진은 AC-MULTI_combo/ 별도 폴더에 저장

4.3 다중 라벨 우선순위 (primary 클래스 결정)

사진 폴더 분류 시 다음 우선순위로 primary 클래스를 결정:

  1. 즉각적 외상 (Critical 외상) 우선 : AC-004 추락 > AC-002 끼임 > AC-005 충돌
  2. 환경 위협 (다수 인명 영향) : AC-001 화재 > AC-008 가스
  3. 쓰러짐 사건 : AC-003 낙상 (생체 이상 동반 시 AC-007 secondary)
  4. 위험구역 진입 : AC-005 또는 AC-008 (단독 시)
  5. 규정 위반 : AC-006 안전모

5. Negative Sample (정상 사진) 구성

정상 사진은 위험 상황 사진과 거의 같은 양(약 50%)이 필요합니다. AI 가 "위험"과 "정상"을 구분하는 법을 배워야 하기 때문입니다.

5.1 정상 사진의 3가지 종류

종류 비율 설명 예시
환경 매칭 정상 50% 위험 사진과 같은 장소·시간대의 정상 작업 사진 AC-001 화재 사진이 B동 2구역에서 찍혔다면, 같은 B동 2구역의 정상 작업 사진을 정상 페어로
헷갈리기 쉬운 정상 (Hard Negative) 35% 위험처럼 보이지만 실제로는 정상인 사진 용접 스파크, 수증기, 자발적 휴식 자세, 의도적 바닥 작업 등 (§2 각 클래스의 "이런 건 라벨링 안 함" 참조)
일반 정상 15% 평범한 작업 사진 일상 작업 사진

5.2 Hard Negative 슬롯 (필수 수집)

각 클래스의 오라벨 방지 사례를 별도 슬롯으로 채워야 합니다.

클래스 Hard Negative 사례 (필수 수집)
AC-001 화재 용접 스파크, 수증기·증기, 석양·조명 반사
AC-002 끼임 자발적 휴식 정지, 공구 조작 중 손 정지, 의도적 고정 작업(볼트 조임)
AC-003 낙상 자발적 웅크림·앉음, 도구 줍기, 의도적 바닥 작업
AC-004 추락 엘리베이터·리프트 하강, 계단 하강, 화물 낙하 (인체 아님)
AC-005 충돌 작업자 구역 우회 통과, 정지 중인 지게차, 유리벽 분리
AC-006 안전모 모자·두건 착용, 의무 구역 외 이동

6. 라벨링 도구 및 파일 포맷

6.1 사용 도구

  • CVAT (Computer Vision Annotation Tool) 또는 발주처가 지정하는 라벨링 환경
  • 도구 접속·계정은 발주처가 제공

6.2 파일 포맷 — COCO Dataset

라벨링 결과는 COCO format JSON 으로 저장합니다. CVAT 에서 Export Task → COCO 1.0 클릭하면 자동 생성됩니다.

COCO 는 라벨링 데이터의 가장 일반적인 표준 형식입니다. 라벨러가 직접 JSON 을 손으로 쓸 필요는 없습니다.

6.3 Bounding Box 좌표 규약

항목 규약
형식 [x_min, y_min, width, height]
단위 픽셀 (절대 좌표)
기준점 이미지 좌상단을 (0, 0) 으로
정규화 사용 안 함 (CVAT 기본값 사용)

예시 : bbox: [320, 180, 210, 155]
→ 이미지의 (320, 180) 위치에서 시작해서 가로 210픽셀 × 세로 155픽셀짜리 박스

6.4 이미지 사양

항목 기준
해상도 최소 1280×720 (HD) 이상
형식 JPEG (.jpg)
색공간 RGB (컬러)

6.5 박스 크기 최소 기준

  • Bounding Box 최소 크기 : 16×16 픽셀
  • 그보다 작은 객체는 박스를 그리지 않음

8. 라벨링 품질 기준 및 검수

PM 은 납품된 라벨링 결과를 다음 기준으로 검수합니다. 통과 못 하면 재작업 요청.

8.1 품질 기준

항목 기준
해상도 모든 이미지 ≥ 1280×720
박스 크기 모든 박스 ≥ 16×16 픽셀
박스 정확도 객체 윤곽에 딱 맞음 (여백 ≤ 5%)
라벨 일치도 같은 사진을 두 사람이 라벨링했을 때 75% 이상 일치
누락 없음 한 사진의 모든 위험 객체에 박스
중복 없음 동일 이미지 여러 번 제출 금지

8.2 거부(재작업) 사유

다음 발견 시 해당 분 전체 거부, 재작업 요청 (재작업 비용 라벨링 업체 부담).

  • 박스가 객체와 동떨어져 있음
  • 클래스 라벨이 명백히 잘못 부착됨 (예: 화재인데 smoke 만, 또는 entrapment 표기)
  • 정상 사진을 위험 상황으로 라벨 (False Positive)
  • 위험 상황을 정상으로 라벨 (False Negative)
  • 얼굴 가림 처리 누락 발견 (§9)
  • 동일 이미지 중복 제출 발견
  • 라벨 스트링이 본 가이드 §1.1 표 외 임의 생성

8.3 이중 라벨링 (품질 보강)

전체 사진의 20% 는 2명의 라벨러가 독립적으로 라벨링합니다.
- 두 결과가 일치 → 정상 통과
- 두 결과가 불일치 → PM(시니어 라벨러) 가 판정

9. PII (개인정보) 처리

본 발주의 모든 이미지에는 얼굴 자동 블러(Blur) 처리 가 이미 적용된 상태로 라벨러에게 제공됩니다.

9.1 라벨러 책임

  • 라벨러는 별도로 얼굴을 가릴 필요 없습니다
  • 다음을 발견하면 PM에게 즉시 보고 :
  • 얼굴 블러가 안 된 사진
  • 이름표·사번이 또렷이 보이는 사진
  • 회사 도면·기밀 모니터 화면이 또렷이 보이는 사진

9.2 작업자 ID 표기

사진에 작업자가 보일 경우 임의 ID 로 부르기 (예: W-042). 실명 사용 금지.

10. FAQ — 헷갈리는 케이스 모음

Q1. 사진에 연기만 있고 불꽃이 안 보여요. 어떤 라벨?

A. smoke 박스만. fire 박스는 만들지 않음.

Q2. 작업자가 바닥에 앉아있어요. 낙상인가요?

A. 자세를 보세요.
- 천천히 앉은 자세 (다리 접고 안정적) → 정상. 라벨 안 함.
- 갑자기 쓰러진 모습 (몸이 비틀려 있거나 한쪽으로 기울어짐) → AC-003.

Q3. 지게차가 작업자 옆 1m 거리로 지나가요. 충돌?

A. 지게차의 이동 방향을 보세요.
- 지게차가 작업자 방향으로 향함 → AC-005 (Critical).
- 지게차가 작업자 옆을 단순 통과 → 정상.

Q4. 안전모는 썼는데 위에 다른 모자를 같이 썼어요.

A. 안전모가 보이면 정상. 라벨 안 함.

Q5. 작업자가 안전모를 손에 들고 있어요.

A. AC-006 미착용 으로 라벨. 들고 있는 건 착용이 아님.

Q6. 사진이 너무 어두워서 위험 상황인지 안 보여요.

A. PM 에게 사진 자체를 거부 요청. 강제로 라벨하지 말 것.

Q7. 작업자가 비틀거리며 걷다가 결국 쓰러져요. AC-007 인가요 AC-003 인가요?

A. 쓰러진 시점부터 AC-003 으로 분류 (낙상 우선). 비틀거리던 부분은 secondary 라벨로 health_anomaly 추가.

Q8. 가스센서 경보가 울렸지만 작업자가 없어요.

A. 라벨 안 함. 작업자가 있을 때만 AC-008.

Q9. 박스를 어디까지 크게 그려야 하나요?

A. 객체의 외곽선에 딱 맞게. 여백은 5% 이내. 너무 크면 재작업.

Q10. 한 사진에 끼임 작업자 1명, 옆에 일하는 정상 작업자 2명이 있어요.

A.
- 끼인 작업자 : person 박스 + 끼인 기계의 machine 박스 + 상황 entrapment
- 정상 작업자 2명 : 박스 없음 (정상이라 라벨링 안 함)

Q11. 사진에 정확히 뭐가 있는지 모르겠어요.

A. PM 에게 문의. 본인 판단으로 라벨링 후 재작업 발생 시 본인 손해.

Q12. CVAT 에서 클래스가 안 보여요.

A. CVAT 관리자에게 문의. 본 가이드 §1.1 의 라벨 스트링이 모두 도구에 등록되어 있어야 함.

11. 작업 기록 및 메모

각 사진의 라벨링 결과에는 다음을 자동 기록 (CVAT 가 자동 처리) :

{
  "labeler_id": "LBL-XXX",          // PM 이 부여한 본인 ID
  "labeled_at": "2026-XX-XXT00:00:00+09:00",
  "review_status": "submitted"
}

라벨러 본인이 직접 추가 메모할 사항이 있으면 labeler_notes.txt 파일에 기록.

12. 작업 분량 및 일정

항목 기준

일일 1인 라벨링 평균 시간 약 25분 / 샘플 (다중 객체 사진 기준)

부록 B. 라벨링 단계별 체크리스트
